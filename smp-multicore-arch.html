📐 架构层级

层级结构总览

典型的现代多核 SMP 处理器（如 ARM Cortex-A72 / A78 / RISC-V SiFive Freedom U740）按以下层次组织：


Core 0 Core 1 Core 2 Core 3 — 4核 SMP
L1 (私有) L2 (私有) L3 (共享) MMU

各层组件

🔹 Core / Hart 内部
  • • 流水线 — Fetch / Decode / Execute / Writeback
  • • MMU — 虚拟地址→物理地址翻译
  • • ITLB — 指令 TLB，常见 32–48 项
  • • DTLB — 数据 TLB，常见 32–48 项
  • • L1 I-Cache — 指令缓存，32–48 KB
  • • L1 D-Cache — 数据缓存，32 KB
  • • L2 Cache — 统一缓存，256 KB–1 MB
🔹 片上互联
  • • L3 Cache — 所有核共享，16–32 MB，分 bank
  • • NoC / Crossbar — 核间互联，AMBA AXI/ACE 协议
  • • Cache Coherency — MESI / MOESI / ARM CHI
🔹 片外
  • • DRAM Controller — DDR4/DDR5，64/128-bit 带宽
  • • 物理内存 — 物理地址空间

🔄 虚拟地址翻译流程

CPU 发起内存访问 → 完整路径
①CPU 生成 虚拟地址 (VA)
②查 ITLB / DTLB — 命中则得 PA
③TLB miss → MMU 查页表
④硬件页表行走器 (HWPTW) 遍历 PTE
⑤将 PTE 写入 TLB，返 PA
⑥用 PA 查 L1 D-Cache
⑦L1 miss → 查 L2 Cache
⑧L2 miss → 查 L3 / 主存
⑨数据返回，填各层 Cache
⑩sfence.vma — 跨 ASID 或 satp 切换时刷 TLB

RISC-V 关键语义

satp 寄存器

MODE=8 (Sv39) / MODE=9 (Sv48) | ASID | PPN

MODE=0 则禁用 Sv39/Sv48 地址翻译

sfence.vma [rs1] [rs2]

rs1=目标虚拟地址（可省略=所有）

rs2=目标 ASID（可省略=所有 ASID）

同步流水线，确保新页表对后续取指可见

ARMv8 对应关系

TTBR0_EL1 → 用户空间页表基址

TTBR1_EL1 → 内核空间页表基址

TCR_EL1 → T0SZ/T1SZ 控制地址宽度

TLBI VMALLIS → 全局 TLB 失效 (VMALL = all)
